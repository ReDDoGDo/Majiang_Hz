# 前端架构文档

## 一、项目概述

本项目是一个基于Godot引擎开发的杭州麻将游戏，采用MVC架构模式，目标平台为Steam（Windows）。

## 二、技术栈

### 2.1 游戏引擎
- **Godot版本**: 4.x
- **渲染管线**: Godot内置Forward+渲染器
- **开发语言**: GDScript (主要) / C# (.NET 6+, 可选)

### 2.2 核心框架
- **UI系统**: Godot内置Control节点系统
- **动画系统**: Godot内置Tween系统
- **网络框架**: Godot高层网络API / SteamP2P (联机功能)
- **Steam集成**: GodotSteam插件

## 四、核心模块说明

### 4.1 核心模块 (Core)

#### GameManager
- **职责**: 游戏生命周期管理
- **功能**: 
  - 游戏初始化
  - 场景切换
  - 游戏状态管理
  - 全局协调

#### EventManager
- **职责**: 事件系统管理
- **功能**:
  - 事件注册/注销
  - 事件触发
  - 模块间通信

#### AudioManager
- **职责**: 音频管理
- **功能**:
  - 背景音乐播放
  - 音效播放
  - 音量控制

#### SaveManager
- **职责**: 存档管理
- **功能**:
  - 游戏进度保存
  - 设置保存
  - 统计数据保存

#### ConfigManager
- **职责**: 配置管理
- **功能**:
  - 配置文件加载
  - 配置参数读取
  - 运行时配置修改

### 4.2 游戏逻辑模块 (Game)

#### TileSystem (麻将牌系统)
- **Tile**: 麻将牌数据结构
- **TileGenerator**: 麻将牌生成器
- **TileWall**: 牌墙管理

#### PlayerSystem (玩家系统)
- **Player**: 玩家基础类
- **PlayerHand**: 手牌管理
- **PlayerAction**: 玩家操作

#### RuleSystem (规则系统)
- **HuChecker**: 胡牌判断
- **FanCalculator**: 番数计算
- **ScoreCalculator**: 计分系统

#### TurnSystem (回合系统)
- **TurnManager**: 回合制管理

#### AISystem (AI系统)
- **AIPlayer**: AI玩家
- **AIDecision**: AI决策
- **AIStrategy**: AI策略

### 4.3 UI模块 (UI)

#### MainMenu (主菜单)
- 游戏启动界面
- 开始游戏按钮
- 设置入口
- 规则说明入口

#### GameUI (游戏界面)
- 牌桌显示
- 手牌显示
- 操作按钮
- 分数显示

#### SettingsUI (设置界面)
- 音量调节
- 游戏速度
- AI难度
- 画面质量

#### ResultUI (结算界面)
- 牌型展示
- 番数详情
- 分数结算

### 4.4 网络模块 (Network) - 可选

#### NetworkManager
- 网络连接管理
- 玩家加入/离开

#### RoomManager
- 房间创建/加入
- 房间列表

#### SyncManager
- 游戏状态同步
- 玩家动作同步

## 五、数据结构

### 5.1 TileData (麻将牌数据)
```gdscript
enum TileType { WAN, TIAO, TONG, FENG, JIAN }

class_name TileData extends Resource
    var id: int
    var type: TileType
    var number: int  # 1-9
    var is_wild_card: bool  # 是否为宝牌
```

### 5.2 PlayerData (玩家数据)
```gdscript
class_name PlayerData extends Resource
    var player_id: int
    var player_name: String
    var hand_tiles: Array[TileData]
    var melded_tiles: Array  # 碰杠牌
    var score: int
    var is_dealer: bool
```

### 5.3 GameData (游戏数据)
```gdscript
class_name GameData extends Resource
    var players: Array[PlayerData]
    var wall_tiles: Array[TileData]
    var wild_card_tile: TileData
    var current_player_index: int
    var current_phase: GamePhase
```

## 六、设计模式

### 6.1 Autoload单例模式
用于全局管理器类（GameManager、EventManager等），通过Godot的Autoload功能实现

### 6.2 观察者模式
用于事件系统，实现模块间解耦

### 6.3 状态模式
用于游戏状态管理（等待、进行、结算等）

### 6.4 策略模式
用于AI决策系统，不同难度使用不同策略

### 6.5 对象池模式
用于麻将牌对象的复用，优化性能

## 七、命名规范

### 7.1 类命名
- PascalCase: GameManager, TileGenerator
- 脚本文件名: snake_case (game_manager.gd, tile_generator.gd)

### 7.2 方法命名
- snake_case: initialize_game, draw_tile
- 信号处理以_on开头: _on_tile_drawn, _on_game_end

### 7.3 变量命名
- 公共变量: snake_case
- 私有变量: _snake_case
- 局部变量: snake_case
- 常量: UPPER_CASE

### 7.4 节点命名
- PascalCase: GameManager, TileWall

## 八、开发流程

### 8.1 场景组织
- main.tscn：主场景，包含Autoload节点
- menu.tscn：主菜单场景
- game.tscn：游戏主场景

### 8.2 场景实例管理
- 所有可复用的游戏对象制作成场景（.tscn）
- 场景按类型分类存放

### 8.3 资源加载
- 使用ResourceLoader.load()加载动态资源
- 使用preload()预加载常用资源

### 8.4 配置管理
- 使用Resource创建配置文件
- 运行时通过ConfigManager读取

## 九、性能优化

### 9.1 对象池
- 麻将牌对象使用对象池管理
- UI元素复用

### 9.2 资源管理
- 及时释放不用的资源
- 使用异步加载

### 9.3 渲染优化
- 使用AtlasTexture
- 合理使用CanvasLayer

### 9.4 代码优化
- 避免_process中的耗时操作
- 使用缓存减少重复计算

- 麻将规则可通过配置文件修改

### 10.3 UI模块化
- UI组件独立封装
- 支持主题切换

### 10.4 网络功能可选
- 网络模块独立封装
