# 前端架构文档

## 一、目录结构

```
src/
├── core/                    # 核心系统

│   │   ├── base_manager.gd    # 管理器基类

│   │   ├── object_pool.gd     # 对象池
│   │   └── utils.gd           # 通用工具函数

├── game/                   # 游戏逻辑
│   ├── tile/               # 麻将牌系统

│   │   ├── tile_data.gd       # 牌数据结构
│   │   ├── tile_factory.gd    # 牌工厂
│   │   └── tile_system.gd     # 牌系统
│   ├── player/             # 玩家系统
│   │   ├── player_data.gd     # 玩家数据
│   │   └── player_system.gd   # 玩家系统
│   ├── rule/               # 规则系统
│   │   ├── rule_type.gd       # 规则类型
│   │   ├── rule_checker.gd    # 规则检查器
│   │   └── rule_system.gd     # 规则系统
│   ├── turn/               # 回合系统
│   │   ├── turn_phase.gd      # 回合阶段
│   │   └── turn_system.gd     # 回合系统
│   └── score/              # 计分系统

│       └── score_system.gd     # 计分系统

│   ├── menu/               # 菜单界面
│   │   └── main_menu.gd       # 主菜单
│   ├── game/               # 游戏界面
│   │   ├── game_ui.gd         # 游戏UI
│   │   └── result_ui.gd       # 结算UI
│   └── settings/           # 设置界面
│       └── settings_ui.gd     # 设置UI
├── network/                # 网络模块
│   ├── network_manager.gd  # 网络管理(Autoload)
│   ├── room_manager.gd     # 房间管理
│   └── sync_manager.gd     # 状态同步
└── data/                   # 数据结构
    ├── game_data.gd        # 游戏数据
    ├── player_state.gd     # 玩家状态
    └── game_config.gd      # 游戏配置
```

## 二、模块特性

### 2.1 核心模块 (Core)

#### BaseManager
- 所有管理器的基类
- 提供初始化和销毁的生命周期管理
- 作为Autoload单例使用

#### GameManager
- 游戏生命周期管理
- 玩家数量控制
- 游戏状态管理
- 信号: game_started, game_ended, state_changed

#### EventManager
- 事件订阅/发布系统
- 观察者模式实现
- 模块间解耦通信

#### AudioManager
- 音乐和音效播放
- 音量控制
- 支持多音轨

#### SaveManager
- JSON格式存档
- 键值对存储
- 自动加载

#### ConfigManager
- 游戏配置管理
- 分层配置系统
- 默认配置

### 2.2 游戏逻辑模块 (Game)

#### TileSystem
- 麻将牌创建和管理
- 洗牌和发牌功能
- 牌池管理

#### PlayerSystem
- 玩家数据管理
- 手牌管理
- 玩家状态跟踪

#### RuleSystem
- 杭州麻将规则实现
- 碰、杠、吃、胡判断
- 胡牌检测算法

#### TurnSystem
- 回合流程控制
- 阶段状态管理
- 回合计数

#### ScoreSystem
- 分数计算
- 番数计算
- 结算处理

### 2.3 UI模块 (UI)

#### BaseUI
- UI基类
- 显示/隐藏管理
- 信号连接模板

#### MainMenu
- 主菜单界面
- 开始游戏、设置、退出按钮

#### GameUI
- 游戏主界面
- 手牌显示
- 操作按钮

#### SettingsUI
- 音量设置
- 显示设置
- 全屏切换

#### ResultUI
- 结算界面
- 分数显示
- 继续游戏

### 2.4 网络模块 (Network)

#### NetworkManager
- ENet网络连接
- 主机/客户端模式
- 数据收发

#### RoomManager
- 房间创建和加入
- 玩家管理
- 房间状态

#### SyncManager
- 游戏状态同步
- 数据序列化
- 网络同步

### 2.5 数据模块 (Data)

#### GameData
- 游戏全局数据
- 回合和轮次信息
- 历史记录

#### PlayerState
- 玩家状态枚举
- 手牌和分数信息

#### GameConfig
- 游戏配置资源
- 可导出配置

## 三、设计模式

### 3.1 MVC架构
- Model: data/, game/模块的数据类
- View: ui/模块的界面类
- Controller: manager/, system类

### 3.2 单例模式
- Autoload管理器
- 全局访问点

### 3.3 观察者模式
- EventManager事件系统
- 信号机制

### 3.4 工厂模式
- TileFactory牌工厂
- 对象创建封装

### 3.5 对象池模式
- ObjectPool对象池
- 频繁创建销毁优化

## 四、命名规范

### 4.1 文件命名
- 类名: PascalCase (例: GameManager.gd)
- 资源文件: snake_case (例: game_config.tres)

### 4.2 代码命名
- 类名: PascalCase
- 方法名: snake_case
- 变量名: snake_case
- 常量: UPPER_CASE
- 私有变量: _snake_case
- 信号: snake_case

### 4.3 节点命名
- PascalCase
- 语义化命名

## 五、信号列表

### 5.1 GameManager信号
- game_started: 游戏开始
- game_ended: 游戏结束
- game_paused: 游戏暂停
- game_resumed: 游戏恢复
- state_changed: 状态改变

### 5.2 EventManager信号
- publish: 发布事件

### 5.3 TileSystem信号
- tile_drawn: 摸牌
- tile_discarded: 出牌
- tiles_shuffled: 洗牌

### 5.4 PlayerSystem信号
- player_joined: 玩家加入
- player_left: 玩家离开
- turn_changed: 回合改变

### 5.5 RuleSystem信号
- meld_available: 可操作
- win_available: 可胡牌

### 5.6 TurnSystem信号
- phase_changed: 阶段改变
- turn_started: 回合开始
- turn_ended: 回合结束

### 5.7 ScoreSystem信号
- score_updated: 分数更新
- score_transferred: 分数转移

### 5.8 NetworkManager信号
- connection_state_changed: 连接状态改变
- player_connected: 玩家连接
- player_disconnected: 玩家断开
- data_received: 数据接收
