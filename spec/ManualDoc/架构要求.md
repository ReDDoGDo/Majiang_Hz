# 杭州麻将项目架构要求

## 一、技术栈

### 1.1 游戏引擎
- **引擎选择**: Godot 4.x
- **开发语言**: GDScript (主要) / C# (.NET 6+, 可选)
- **渲染管线**: Godot内置Forward+渲染器

### 1.2 开发工具
- **IDE**: Godot Editor / VS Code (with Godot插件)
- **版本控制**: Git
- **项目管理**: GitHub Projects / Trello
- **设计工具**: Figma / Adobe XD (UI设计)

### 1.3 第三方库
- **网络框架**: Godot高层网络API / SteamP2P (联机功能)
- **Steam集成**: GodotSteam插件
- **UI框架**: Godot内置Control节点系统
- **动画系统**: Godot内置Tween系统
- **对象池**: 自研或使用Godot对象池方案

### 1.4 目标平台
- **主要平台**: Steam (Windows)
- **可选平台**: macOS, Linux

## 二、架构设计原则

### 2.1 核心架构模式
- **设计模式**: MVC (Model-View-Controller)
- **模块化设计**: 高内聚、低耦合
- **事件驱动**: 使用观察者模式进行模块间通信
- **单一职责**: 每个类只负责一个功能

### 2.2 代码组织原则
- **命名空间规范**: MajiangHZ.[模块名].[子模块]
- **命名规范**: 
  - 类名/脚本名: PascalCase (例: GameManager)
  - 方法名: snake_case (例: initialize_game) - GDScript
  - 变量名: snake_case (例: player_count) - GDScript
  - 常量: UPPER_CASE (例: MAX_PLAYER_COUNT)
  - 私有变量: _snake_case (例: _current_player) - GDScript
  - 节点名: PascalCase (例: GameManager)
- **注释规范**: 所有公共方法必须有文档注释

### 2.3 资源管理
- **资源路径**: res://[资源类型]/
- **资源命名**: [类型]_[名称]_[变体] (例: Tile_Wan_1)
- **资源加载**: 使用ResourceLoader.load()或preload()
- **内存管理**: 及时释放不用的资源，使用对象池

## 四、模块划分

### 4.1 核心模块 (Core)
- **GameManager**: 游戏生命周期管理 (Autoload单例)
- **EventManager**: 事件系统 (Autoload单例)
- **AudioManager**: 音频管理 (Autoload单例)
- **SaveManager**: 存档管理 (Autoload单例)
- **ConfigManager**: 配置管理 (Autoload单例)

### 4.2 游戏逻辑模块 (Game)
- **TileSystem**: 麻将牌系统 (场景+脚本)
- **PlayerSystem**: 玩家系统 (场景+脚本)
- **RuleSystem**: 规则系统 (资源+脚本)
- **TurnSystem**: 回合系统 (场景+脚本)
- **ScoreSystem**: 计分系统 (资源+脚本)

### 4.3 UI模块 (UI)
- **MainMenu**: 主菜单 (场景)
- **GameUI**: 游戏界面 (场景)
- **SettingsUI**: 设置界面 (场景)
- **ResultUI**: 结算界面 (场景)

### 4.4 网络模块 (Network)
- **NetworkManager**: 网络管理 (Autoload单例)
- **RoomManager**: 房间管理 (场景+脚本)
- **SyncManager**: 状态同步 (场景+脚本)

## 五、开发规范

### 5.1 代码规范
- 所有脚本继承自Godot节点类(Node, Node2D, Control等)
- 避免在_process中使用get_node等耗时操作，使用@onready缓存
- 使用对象池管理频繁创建销毁的对象
- 所有公共方法必须有错误处理
- 避免使用全局变量，使用Autoload单例或依赖注入
- 使用类型提示提高代码可读性

### 5.2 Git提交规范
- **feat**: 新功能
- **fix**: 修复bug
- **refactor**: 重构代码
- **docs**: 文档更新
- **style**: 代码格式调整
- **test**: 测试相关
- **chore**: 构建/工具链相关

提交信息格式: `[类型] 简短描述`
例: `[feat] 添加麻将牌翻转动画`

### 5.3 版本管理
- **主版本号**: 重大功能更新或架构变更
- **次版本号**: 新功能添加
- **修订号**: Bug修复和小优化
- 版本号格式: v1.0.0

## 六、性能要求

### 6.1 帧率要求
- 目标帧率: 60 FPS
- 最低帧率: 30 FPS

### 6.2 内存要求
- 初始内存占用: < 200MB
- 游戏中内存占用: < 500MB

### 6.3 加载时间
- 首次启动: < 5秒
- 场景切换: < 2秒

## 七、发布要求

### 7.1 Steam发布
- 集成GodotSteam插件
- 实现Steam成就系统
- 支持Steam云存档
- 添加Steam排行榜（可选）

### 7.2 安装包要求
- Windows安装包: < 500MB
- 支持Windows 10及以上版本
- 使用Godot导出模板打包
