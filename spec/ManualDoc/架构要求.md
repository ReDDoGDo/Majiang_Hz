# 杭州麻将项目架构要求

## 一、技术栈

### 1.1 游戏引擎
- **引擎选择**: Unity 2022.3 LTS
- **开发语言**: C# (.NET Standard 2.1)
- **渲染管线**: URP (Universal Render Pipeline)

### 1.2 开发工具
- **IDE**: Visual Studio 2022 / VS Code
- **版本控制**: Git
- **项目管理**: GitHub Projects / Trello
- **设计工具**: Figma / Adobe XD (UI设计)

### 1.3 第三方库
- **网络框架**: Photon PUN2 (联机功能)
- **Steam集成**: Steamworks.NET
- **UI框架**: UGUI (Unity内置)
- **动画系统**: DOTween (可选)
- **对象池**: 自研或使用现有方案

### 1.4 目标平台
- **主要平台**: Steam (Windows)
- **可选平台**: macOS, Linux

## 二、架构设计原则

### 2.1 核心架构模式
- **设计模式**: MVC (Model-View-Controller)
- **模块化设计**: 高内聚、低耦合
- **事件驱动**: 使用观察者模式进行模块间通信
- **单一职责**: 每个类只负责一个功能

### 2.2 代码组织原则
- **命名空间规范**: MajiangHZ.[模块名].[子模块]
- **命名规范**: 
  - 类名: PascalCase (例: GameManager)
  - 方法名: PascalCase (例: InitializeGame)
  - 变量名: camelCase (例: playerCount)
  - 常量: UPPER_CASE (例: MAX_PLAYER_COUNT)
  - 私有变量: _camelCase (例: _currentPlayer)
- **注释规范**: 所有公共方法必须有XML注释

### 2.3 资源管理
- **资源路径**: Assets/Resources/[资源类型]/
- **资源命名**: [类型]_[名称]_[变体] (例: Tile_Wan_1)
- **资源加载**: 使用Resources.Load或Addressables
- **内存管理**: 及时释放不用的资源，使用对象池

## 四、模块划分

### 4.1 核心模块 (Core)
- **GameManager**: 游戏生命周期管理
- **EventManager**: 事件系统
- **AudioManager**: 音频管理
- **SaveManager**: 存档管理
- **ConfigManager**: 配置管理

### 4.2 游戏逻辑模块 (Game)
- **TileSystem**: 麻将牌系统
- **PlayerSystem**: 玩家系统
- **RuleSystem**: 规则系统
- **TurnSystem**: 回合系统
- **ScoreSystem**: 计分系统

### 4.3 UI模块 (UI)
- **MainMenu**: 主菜单
- **GameUI**: 游戏界面
- **SettingsUI**: 设置界面
- **ResultUI**: 结算界面

### 4.4 网络模块 (Network)
- **NetworkManager**: 网络管理
- **RoomManager**: 房间管理
- **SyncManager**: 状态同步

## 五、开发规范

### 5.1 代码规范
- 所有类继承自MonoBehaviour时，使用RequireComponent特性
- 避免在Update中使用Find、GetComponent等耗时操作
- 使用对象池管理频繁创建销毁的对象
- 所有公共接口必须有错误处理
- 避免使用全局变量，使用单例模式或依赖注入

### 5.2 Git提交规范
- **feat**: 新功能
- **fix**: 修复bug
- **refactor**: 重构代码
- **docs**: 文档更新
- **style**: 代码格式调整
- **test**: 测试相关
- **chore**: 构建/工具链相关

提交信息格式: `[类型] 简短描述`
例: `[feat] 添加麻将牌翻转动画`

### 5.3 版本管理
- **主版本号**: 重大功能更新或架构变更
- **次版本号**: 新功能添加
- **修订号**: Bug修复和小优化
- 版本号格式: v1.0.0

## 六、性能要求

### 6.1 帧率要求
- 目标帧率: 60 FPS
- 最低帧率: 30 FPS

### 6.2 内存要求
- 初始内存占用: < 200MB
- 游戏中内存占用: < 500MB

### 6.3 加载时间
- 首次启动: < 5秒
- 场景切换: < 2秒

## 七、发布要求

### 7.1 Steam发布
- 集成Steamworks SDK
- 实现Steam成就系统
- 支持Steam云存档
- 添加Steam排行榜（可选）

### 7.2 安装包要求
- Windows安装包: < 500MB
- 支持Windows 10及以上版本

### 8.3 性能测试
